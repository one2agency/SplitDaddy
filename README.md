# 💸 SplitDaddy

[![tests](https://github.com/one2agency/SplitDaddy/actions/workflows/tests.yml/badge.svg)](https://github.com/one2agency/SplitDaddy/actions/workflows/tests.yml)

> 📐 Як це працює всередині та яку логіку закладено — див. [ARCHITECTURE.md](ARCHITECTURE.md).

Telegram-бот для поділу витрат після вечірки. Працює у **груповому чаті**:
учасники додають витрати (хто заплатив і за кого), а бот рахує, **хто кому
скільки винен** з мінімальною кількістю переказів.

Без слеш-команд і перемикання розкладки — усе через **кнопки українською**
та **текстовий ввід витрат** (сума на початку повідомлення).

## Стек

- Python 3.11+, [aiogram 3.x](https://docs.aiogram.dev/), SQLite (`aiosqlite`)
- Режим: long polling
- Гроші — в **копійках** (integer), валюта одна — `UAH`

## Запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env        # вписати BOT_TOKEN від @BotFather
python -m bot.main
```

| Змінна      | Опис                            | За замовч.       |
|-------------|---------------------------------|------------------|
| `BOT_TOKEN` | Токен від @BotFather (обов'язк.)| —                |
| `DB_PATH`   | Шлях до файлу SQLite            | `splitdaddy.db`  |

### ⚠️ Privacy Mode = OFF (обов'язково)

Бот реагує на звичайні повідомлення (кнопки й витрати), а не на слеш-команди.
У @BotFather: `/setprivacy` → обери бота → **Disable**. Після зміни **видали й
знову додай** бота в групу. Бот ігнорує балачки, фото й меми — реагує лише на
кнопки та повідомлення, що починаються з числа.

> Альтернатива при Privacy ON: ті самі повідомлення-витрати слати **у відповідь
> (reply)** на повідомлення бота — реплаї доходять і при ввімкненій приватності.

## Як користуватись

Коли бота додають у групу, він показує **клавіатуру з кнопками**:

| Кнопка | Дія |
|---|---|
| 🎉 **Нова вечірка** | Почати сесію; хто натиснув — організатор |
| 👥 **Учасники** | Список учасників + як додати |
| 📊 **Підсумок** | Поточний розрахунок (будь-коли) + кнопки 🗑 |
| ✅ **Завершити** | Фінальний підсумок, сесія закривається |
| ❓ **Допомога** | Короткий гайд |

### Додавання витрати (текстом)

```
<сума> <опис> [@користувачі...] [-я]
```

- `1200 піца` — поділ на **всіх** учасників вечірки.
- `350 коктейлі @ira @oleh` — поділ на відмічених **+ автора**.
- `800 таксі @ira -я` — автор заплатив, але `-я` виключає його з поділу.

Сума: `800`, `800.50` або `800,50`. Опис — текст між сумою та першим тегом.
Платник — завжди автор повідомлення. Автор витрати автоматично стає учасником;
відмічені теги — теж додаються в учасники.

### Учасники

- **Організатор** — хто створив вечірку.
- «🙋 Я в ділі» — inline-кнопка під вітанням, щоб приєднатись самому.
- Організатор/адмін: додати — `+ @user`, прибрати — `- @user` (доки в людини
  немає витрат).
- Люди без @username ідентифікуються через `text_mention` (id у entity) — їм
  достатньо хоч раз проявитись у чаті чи натиснути «Я в ділі».

### Видалення витрати

Кнопка 🗑 під «📊 Підсумок», або текст `видалити BX-3`. Доступно автору
витрати або адміну чату.

## ID витрат

Кожна вечірка має короткий глобально-унікальний **код** (2–4 символи Base32 без
схожих 0/O, 1/I), напр. `BX`. Витрати — `#BX-1`, `#BX-2`, … (порядковий номер у
межах вечірки). Тож паралельні вечірки не плутаються.

## Алгоритм поділу

1. За замовчуванням витрата ділиться **порівну на всіх учасників**; з тегами —
   на відмічених + автора; `-я` виключає автора.
2. Частка = `amount_cents / N бенефіціарів`; залишок копійок — першим
   бенефіціарам (сума часток точно = сумі витрати).
3. `net = paid − owed`. `net > 0` — кредитор, `< 0` — боржник. Сума всіх = 0.
4. **Мінімізація переказів:** жадібно — найбільший боржник ↔ найбільший
   кредитор, поки баланси не занеуляться.

Уся логіка — у [`bot/splitter.py`](bot/splitter.py), без залежностей від
Telegram чи БД.

## Структура

```
bot/
  config.py      — конфіг з env
  db.py          — шар даних (aiosqlite): коди, seq, учасники, транзакції
  splitter.py    — чиста логіка поділу й мінімізації переказів
  parsing.py     — розбір витрат/команд з message entities
  keyboards.py   — reply- та inline-клавіатури
  formatting.py  — рендеринг повідомлень (HTML)
  handlers.py    — кнопки, текстовий роутер, callbacks, вхід у групу
  main.py        — точка входу + middleware реєстрації користувачів
tests/
  test_splitter.py, test_parsing.py
```
